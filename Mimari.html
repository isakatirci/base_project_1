Mimari Proje Haritası (Kapsamlı Akış)

Bu sayfa, dışarıdan gelen bir kullanıcının sisteme nasıl girdiğini ve isteklerin mikroservisler arasında yukarıdan aşağıya (Top-Down) nasıl çağrıldığını net bir şekilde açıklar.

📱 Mobil / Web Kullanıcı

Son Kullanıcı İstemcileri (React, iOS)

HTTP/HTTPS (Dış Trafik)

🌐 K8s Ingress & Istio Gateway

Ağ Sınır Kapısı (Border Layer): Dış internetten gelen tüm trafiğin Bulut Ağına girdiği noktadır. İstekleri şifreler ve API Gateway'e gönderir.

Ağ Yönlendirmesi

🚪 API Gateway

Yönlendirme (Routing): Mikroservislerin tek giriş merkezidir. Sisteme kimlerin girebileceğini Auth Server'a sorar.

Token Doğrulama

🔐 Authorization Server

Kimlik Doğrulama: Kullanıcıların OAuth2 Token (Oturum Bileti) yetkilerini denetler.

Güvenli İstek (İç Trafik)

🧩 Product Composite Service

Toplayıcı (Aggregator): Müşterinin tek ekranda göreceği Ürün, Yorum ve Önerileri bir araya toplamak için alttaki 3 alt servisi aynanda çağırır. (Resilience4j ile Çökme Koruması vardır).

⚙️ Config Server

Uygulamaların gizli şifrelerini (application.yml) içindeki Config Depositinden (Git) çekerek anında tüm üstteki servislere dağıtır.

📊 Prometheus & Grafana

Mimarinin tamamını yukarıdan izler; tüm Java, Veritabanı ve Kafka sunucularının Anlık CPU ve RAM Metriklerini (Dashboard) sağlar.

📝 EFK Stack (Günlük/Log)

Elasticsearch, Fluentd ve Kibana: Tüm mikroservislerden düşen text hatalarını arkaplanda toparlar, merkezileştirir ve arama olanağı sunar.
